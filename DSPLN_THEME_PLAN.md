# DSPLN Shopify Theme Plan

## Project Name

`dspln-shopify-theme`

This theme is a Dawn-based Shopify theme for DSPLN configurator products.

It should not use or copy the current live theme. Dawn is the base because it is Shopify's official reference theme and already provides the native Shopify theme structure we need.

## Role

The theme owns Shopify storefront presentation:

- Product templates
- Liquid sections and snippets
- Cart line item rendering
- Theme CSS
- Account page surfaces
- Theme-specific JavaScript

The theme does not own configurator business logic.

`dspln-configurator-platform` owns:

- 3D configurator UI and state
- Product-family behavior
- Uploads and preview generation
- Save/load behavior
- Add-to-cart payloads
- Shopify line item property data contract

## Stage 1 Goal

Create a clean unpublished Shopify theme that can host DSPLN configurator products without fighting the current live theme's CSS, product templates, or cart markup.

## Initial Product Template

`templates/product.dspln-configurator.json`

This template renders:

- Product identity from Shopify
- One controlled configurator iframe section
- A small fallback link if the iframe does not load

The configurator URL is controlled by section settings so each test product can point to the correct platform route.

## Non-Negotiables

- Do not publish this theme without explicit approval.
- Do not edit the current live theme from this repo.
- Do not move configurator logic into Liquid.
- Do not change live product handles, variants, prices, or checkout property names without explicit approval.
- Keep Dawn's license intact.

4. Add test theme instructions once Shopify CLI is installed and connected.

## Shared Cart Renderer

`snippets/dspln-line-item-properties.liquid` renders customer-facing DSPLN customization summaries for the cart page and cart drawer.

The renderer should:

- show product preview imagery when a preview image property is available
- group known Gi properties into Kimono, Belt, Pant, and Logos
- show unrecognized customer-facing properties under Additional options
- hide internal, production, tech-pack, design JSON, preview, and underscore-prefixed keys from customer-facing cart UI
- leave the original line item properties intact for checkout, orders, and backend/admin workflows
